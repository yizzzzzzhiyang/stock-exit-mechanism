# 股票退出机制 v1.1 操作手册

> Serenity 1.0 选股 + 股票退出机制 1.1 QMT 执行

---

## 系统架构

```
收盘后:  Serenity 1.0  →  serenity_picks.json
                             ↓
盘前:    QMT 读取 JSON  →  自动更新股票池
盘中:    CUSUM 择时     →  passorder 买入
         追损3.0σ 退出  →  passorder 卖出
```

---

## 每日操作流程

### Step 1: 收盘后跑 Serenity（15:30，手动）

```bash
# 确保 NAS 已挂载（Finder 里能看到 /Volumes/quant）
cd ~/.hermes/projects/serenity-report/scripts
python3 pipeline.py
```

**预期输出：**
```
A: 板块筛选 + AI...
  ✅ 化学制药
  ✅ 创新药
B: Serenity...
C: Wyckoff...
✅ /Users/yizhiyang/public/pipeline_report.html
📋 QMT选股清单: /Users/yizhiyang/public/serenity_picks.json (5只)
```

**产物：**
- `~/public/pipeline_report.html` — 给你看的完整报告
- `~/public/serenity_picks.json` — 给 QMT 读的选股清单

### Step 2: 检查报告（15:35，30秒）

打开 `~/public/pipeline_report.html`，快速确认：
1. 板块分析是否合理（涨幅、成交额、AI 解读）
2. 个股 GARP 评分是否靠谱
3. 有没有明显有问题的股票（ST、次新、涨跌停）

**如果发现明显问题 → 在 QMT 里临时禁用对应股票（可选），或者相信 Serenity 的判断。**

### Step 3: QMT 自动运行（次日 9:25 起）

QMT 策略在 9:25 自动执行：
```
[股票退出机制 v1.1] 初始化完成
  股票池(5只): ['688578.SH', '300573.SZ', ...]
  [选股] 来自 Serenity 2026-07-05 — 5只
  追损参数: lower=1.0σ  trail=3.0σ  max_hold=20d
```

如果 JSON 不可用：
```
[选股] serenity_picks.json 不存在，使用默认股票池
  股票池(5只): ['002714.SZ', '601318.SH', ...]
```

之后 QMT 自动执行全天交易，无需干预。

## Serenity 不可用时的降级方案

| 场景 | QMT 行为 |
|------|------|
| NAS 没挂载，pipeline 跑不了 | JSON 不存在 → 回退到默认 5 只固定池 |
| pipeline 跑了但出错 | JSON 未生成 → 回退到默认 5 只 |
| JSON 超过 3 天未更新 | 过期 → 回退到默认 5 只 |
| JSON 存在但内容为空 | 空列表 → 回退到默认 5 只 |

默认池（`fallback_pool`）为牧原、平安、五粮液、三生国健、东方电气。
可在 `CONFIG['fallback_pool']` 中修改。

## QMT 策略配置

### 必须修改的配置（1 处）

在 QMT 策略代码中找到：
```python
CONFIG = {
    'account': '你的资金账号',   # ← 改成你的真实账号
```

其余参数可保持默认。

### 可选修改的参数

| 参数 | 默认值 | 作用 |
|------|:---:|------|
| `cash_per_stock` | 10000 | 每只买入金额 |
| `max_positions` | 5 | 最大持仓数 |
| `lower_mult` | 1.0 | 下屏障倍数（止损灵敏度） |
| `trail_mult` | 3.0 | 追损线倍数（止盈灵敏度） |
| `max_hold_days` | 20 | 最大持仓天数 |
| `daily_loss_limit` | -0.05 | 单日亏损熔断 (-5%) |

## 安装步骤（首次）

### 1. QMT 补充数据

在 QMT 客户端 → 数据管理 → 补充历史数据：
- 拉取所有 A 股的日线数据（或至少沪深 300）

### 2. 创建 QMT 策略

模型交易 → 新建策略 → 粘贴 `股票退出机制_v1.1_QMT.py` 全部代码 → 修改 `account` → 保存

### 3. 绑定模拟账户

先跑模拟盘 2 周，确认：
- 入场信号频率正常（每天 0-2 次入场）
- 成交正常（无连续废单）
- 卖出逻辑正常（止损/追损/时间三种退出都有发生）

### 4. 切换到实盘

确认无误后，切换到实盘账户。

## 常见问题

### Q: NAS 挂载不上怎么办？
A: 手动 Finder → 前往 → 连接服务器 → `smb://192.168.5.41/quant`。

### Q: Serenity 总是选同一批股票，没变化？
A: 正常。Serenity 基于基本面+供应链分析，风格偏中长线。同一板块连续出现在榜单上是合理的——说明趋势在持续。

### Q: QMT 提示 "serenity_picks.json 不存在"？
A: 说明前一交易日没跑 pipeline。QMT 会自动回退到默认股票池，不影响当天交易。收盘后补跑 Pipeline 即可。

### Q: 想临时加一只股票进池？
A: 两种方式：
1. 加在 `CONFIG['fallback_pool']` 里（下次 QMT 重启生效）
2. Serenity pipeline 停用当天，QMT 会使用 `fallback_pool`

### Q: 想临时停止交易某只股票？
A: 在 `check_entries` 函数里加一行：
```python
if stock == '688XXX.SH': continue
```
